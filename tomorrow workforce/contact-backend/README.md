# Contact Backend (Next.js + Supabase)

This service receives contact form submissions and stores them in Supabase.

## 1. Create Supabase table

Run this SQL in Supabase SQL editor:

```sql
create table if not exists public.contact_submissions (
  id bigint generated by default as identity primary key,
  name text not null,
  email text not null,
  company text,
  use_case text,
  budget text,
  workflow text,
  source_page text,
  submitted_at timestamptz not null default now()
);
```

## 2. Configure environment variables

Copy `.env.example` to `.env.local` and fill values:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `ALLOWED_ORIGIN` (your frontend domain)

## 3. Run locally

```bash
npm install
npm run dev
```

API endpoint:

- `POST /api/contact`

## 4. Deploy on Render

Create a Web Service from this folder (`contact-backend`) with:

- Build command: `npm install && npm run build`
- Start command: `npm run start`

Set env vars in Render:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `ALLOWED_ORIGIN`

## 5. Connect static contact form

In `contact.html`, set:

```html
<form data-endpoint="https://your-render-service.onrender.com/api/contact">
```

The frontend script already posts JSON to this endpoint.
